# Trip Notes Feature Plan

## Overview

Trip Notes allows users to create geotagged, timestamped annotations during active trips. Notes are anchored to specific locations and moments, enriching trip data with personal context. They appear as markers on maps, entries in trip timelines, and are included in exported trip files.

## User Stories

1. As a user, I want to quickly add a text note during an active trip without pausing tracking
2. As a user, I want to attach a photo to a note so I can remember what I saw at a location
3. As a user, I want to record a voice memo while cycling/driving without touching the screen
4. As a user, I want to categorize notes (scenic, food, hazard, rest, fuel, POI) for easy filtering
5. As a user, I want to see my notes as markers on the trip map (live and in trip detail)
6. As a user, I want to add notes retroactively by tapping on the trip route after completion
7. As a user, I want notes included in my exported trip files (.traq, GPX waypoints, GeoJSON)
8. As a user, I want the AI to auto-generate notes for significant events (long stops, mode changes)

## Architecture

### Module Placement

- `core/data` — TripNoteEntity, TripNoteDao, TripNoteRepository
- `core/common` — NoteType, NoteCategory enums
- `core/ai` — AutoNoteGenerator (detects stop events, mode changes)
- `core/export` — Include notes as GPX waypoints / GeoJSON features / .traq entries
- `feature/tracking` — Note creation UI during active trip
- `feature/trip-detail` — Note display on map + timeline, retroactive note addition
- `core/maps` — Note marker rendering on map

### Data Flow

```
User taps "Add Note" on tracking screen
  → NoteCreationSheet opens (bottom sheet)
  → User types text / takes photo / records voice
  → ViewModel calls tripNoteRepository.insert(note)
  → Note persisted to Room with current GPS coordinates + timestamp
  → Map updates to show note marker
  → Export includes note when trip is exported
```

### Auto-Generated Notes

```
TrackingService detects:
  - Stop duration > 3 min → auto-note "Stopped for X minutes"
  - Transport mode change → auto-note "Switched to [mode]"
  - Significant elevation change → auto-note "Climbed X meters"
  
AutoNoteGenerator:
  - Injected into TrackingService
  - Analyzes TrackPoint stream + transport mode changes
  - Emits TripNote objects to TripNoteRepository
```

## UI Design

### Tracking Screen — Note Creation

- **Entry point**: Small circular FAB (bottom-right of map, above the bottom sheet)
  with a "+" or "note" icon
- **Quick actions**: Tap = text note, long-press = show options (text/photo/voice)
- **Bottom sheet**: Slides up with:
  - Text field (auto-focused, with current location shown)
  - Category chips (horizontal scroll: Scenic, Food, Hazard, Rest, Fuel, POI)
  - Attach buttons: Camera, Gallery, Microphone
  - Save button
- **Voice recording**: Full-screen overlay with waveform animation, tap to stop
- **Photo capture**: Launch system camera, return to note sheet with preview

### Trip Detail Screen — Note Display

- **Map markers**: Custom markers for each note type (colored by category)
  - Tap marker → info window with note preview
  - Tap info window → expand to full note detail
- **Timeline view**: Scrollable list interleaving track segments and notes
  - Notes show: icon, category chip, text preview, timestamp, photo thumbnail
- **Retroactive add**: Tap on route polyline → "Add note at this location" option

### Note Detail View

- Full text display
- Full-size photo with zoom/pan
- Audio playback with waveform
- Edit / delete options
- Location shown on mini-map
- Timestamp and category

## Export Integration

### .traq Format

```json
{
  "notes": [
    {
      "id": "uuid",
      "timestamp": "2026-02-27T10:30:00Z",
      "latitude": 37.422,
      "longitude": -122.084,
      "text": "Great viewpoint!",
      "type": "TEXT",
      "category": "SCENIC",
      "mediaUri": null,
      "isAutoGenerated": false
    }
  ]
}
```

### GPX Format

Notes export as `<wpt>` waypoints:
```xml
<wpt lat="37.422" lon="-122.084">
  <time>2026-02-27T10:30:00Z</time>
  <name>Great viewpoint!</name>
  <desc>Category: Scenic</desc>
  <type>SCENIC</type>
</wpt>
```

### GeoJSON Format

Notes export as Point features in the FeatureCollection:
```json
{
  "type": "Feature",
  "geometry": { "type": "Point", "coordinates": [-122.084, 37.422] },
  "properties": {
    "type": "note",
    "text": "Great viewpoint!",
    "category": "SCENIC",
    "timestamp": "2026-02-27T10:30:00Z"
  }
}
```

## Implementation Phases

### Phase 1: Foundation (Text Notes)
- Add TripNote entity, DAO, repository to core/data
- Add NoteType, NoteCategory enums to core/common
- Add note creation bottom sheet to tracking screen
- Add note markers to map (both tracking and trip detail)
- Basic text notes with category selection
- Room migration for trip_notes table

### Phase 2: Media Notes
- Photo capture and gallery picker integration
- Thumbnail generation and storage
- Voice recording with MediaRecorder
- Audio playback in note detail
- File management (storage, cleanup on trip delete)

### Phase 3: AI Auto-Notes
- AutoNoteGenerator in core/ai
- Stop detection → auto-note
- Transport mode change → auto-note
- Elevation milestone → auto-note
- Auto-notes marked with distinct icon/style

### Phase 4: Export + Retroactive
- Include notes in .traq export
- Include notes as GPX waypoints
- Include notes as GeoJSON Point features
- Retroactive note creation on trip detail map
- Note timeline view in trip detail

### Phase 5: Voice Transcription
- On-device speech-to-text (Android SpeechRecognizer)
- Transcription stored alongside audio file
- Searchable note text from voice memos

## Database Migration

```kotlin
val MIGRATION_X_Y = object : Migration(X, Y) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("""
            CREATE TABLE IF NOT EXISTS trip_notes (
                id TEXT NOT NULL PRIMARY KEY,
                tripId TEXT NOT NULL,
                timestamp INTEGER NOT NULL,
                latitude REAL NOT NULL,
                longitude REAL NOT NULL,
                text TEXT,
                type TEXT NOT NULL,
                category TEXT,
                mediaUri TEXT,
                thumbnailUri TEXT,
                transcription TEXT,
                isAutoGenerated INTEGER NOT NULL DEFAULT 0,
                segmentIndex INTEGER NOT NULL DEFAULT 0,
                FOREIGN KEY (tripId) REFERENCES trips(id) ON DELETE CASCADE
            )
        """)
        db.execSQL("CREATE INDEX IF NOT EXISTS index_trip_notes_tripId ON trip_notes(tripId)")
    }
}
```

## Acceptance Criteria

- [ ] User can create a text note during an active trip in under 3 taps
- [ ] Notes appear as markers on the live tracking map
- [ ] Notes appear as markers on the trip detail map
- [ ] Notes are included in .traq, GPX, and GeoJSON exports
- [ ] Photo notes capture and display correctly
- [ ] Voice notes record and play back correctly
- [ ] Auto-notes generate for stops > 3 min and mode changes
- [ ] Notes persist across app restarts
- [ ] Deleting a trip cascades to delete its notes
- [ ] Retroactive notes can be added from trip detail
